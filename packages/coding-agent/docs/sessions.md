# 会话

Pi 将会话保存下来，以便你可以继续工作、从之前的轮次分支，以及回溯之前的路径。

## 会话存储

会话自动保存到 `~/.pi/agent/sessions/`，按工作目录组织。每个会话是一个 JSONL 文件，具有树形结构。

```bash
pi -c                  # 继续最近的会话
pi -r                  # 浏览并选择历史会话
pi --no-session        # 临时模式，不保存
pi --session <路径|ID>  # 使用特定的会话文件或部分会话 ID
pi --fork <路径|ID>    # 将会话文件或部分会话 ID 分支到一个新会话
```

在交互模式下使用 `/session` 查看当前会话文件、会话 ID、消息数量、token 数和花费。

关于 JSONL 文件格式和 SessionManager API，请参阅[会话格式](zh/session-format.md)。

## 会话命令

| 命令 | 描述 |
|---------|-------------|
| `/resume` | 浏览并选择之前的会话 |
| `/new` | 开始新会话 |
| `/name <名称>` | 设置当前会话的显示名称 |
| `/session` | 显示会话信息 |
| `/tree` | 导航当前会话树 |
| `/fork` | 从之前的用户消息创建新会话 |
| `/clone` | 将当前活动分支复制到新会话 |
| `/compact [提示]` | 总结较旧的上下文；参阅[压缩](zh/compaction.md) |
| `/export [文件]` | 将会话导出为 HTML |
| `/share` | 以私有 GitHub Gist 形式上传并生成可分享的 HTML 链接 |

## 恢复与删除会话

`/resume` 打开当前项目的交互式会话选择器。`pi -r` 在启动时打开相同的选择器。

在选择器中你可以：

- 通过输入进行搜索
- 使用 Ctrl+P 切换路径显示
- 使用 Ctrl+S 切换排序模式
- 使用 Ctrl+N 过滤已命名的会话
- 使用 Ctrl+R 重命名
- 使用 Ctrl+D 删除，然后确认

如果可用，pi 会使用 `trash` 命令行工具进行删除，而不是永久移除文件。

## 命名会话

使用 `/name <名称>` 设置可读的会话名称：

```text
/name 重构认证模块
```

已命名的会话在 `/resume` 和 `pi -r` 中更容易找到。

## 使用 `/tree` 进行分支

会话以树形结构存储。每条记录都有 `id` 和 `parentId`，当前位置是活动叶子节点。`/tree` 允许你跳转到之前的任意位置并继续，而无需创建新文件。

<p align="center"><img src="../images/tree-view.png" alt="树形视图" width="600"></p>

示例结构：

```text
├─ user: "你好，你能帮忙..."
│  └─ assistant: "当然！我可以..."
│     ├─ user: "我们试试方案 A..."
│     │  └─ assistant: "对于方案 A..."
│     │     └─ user: "成功了..."  ← 当前活动
│     └─ user: "实际上，方案 B..."
│        └─ assistant: "对于方案 B..."
```

### 树形控件

| 按键 | 操作 |
|-----|--------|
| ↑/↓ | 导航可见记录 |
| ←/→ | 向上/向下翻页 |
| Ctrl+←/Ctrl+→ 或 Alt+←/Alt+→ | 折叠/展开或在分支段之间跳转 |
| Shift+L | 在选中的记录上设置或清除标签 |
| Shift+T | 切换标签时间戳显示 |
| Enter | 选择记录 |
| Escape/Ctrl+C | 取消 |
| Ctrl+O | 循环切换过滤模式 |

过滤模式包括：默认、无工具、仅用户、仅标签、全部。通过[设置](zh/settings.md)中的 `treeFilterMode` 配置默认模式。

### 选择行为

选择用户或自定义消息：

1. 将叶子节点移动到所选消息的父节点。
2. 将所选消息的文本放入编辑器中。
3. 让你编辑并重新提交，从而创建新分支。

选择助手、工具、压缩或其他非用户记录：

1. 将叶子节点移动到该记录。
2. 编辑器保持为空。
3. 让你从该点继续。

选择根用户消息会将叶子节点重置为空对话，并将原始提示放入编辑器。

## `/tree`、`/fork` 和 `/clone`

| 功能 | `/tree` | `/fork` | `/clone` |
|---------|---------|---------|----------|
| 输出 | 同一会话文件 | 新会话文件 | 新会话文件 |
| 视图 | 完整树形 | 用户消息选择器 | 当前活动分支 |
| 典型用途 | 在原地探索替代方案 | 从之前的提示开始新会话 | 在继续之前复制当前工作 |
| 摘要 | 可选分支摘要 | 无 | 无 |

当你想把替代方案保留在一起时使用 `/tree`。当你想要一个单独的会话文件时使用 `/fork` 或 `/clone`。

## 分支摘要

当 `/tree` 从一个分支切换到另一个分支时，pi 可以总结被放弃的分支，并将该摘要附加到新位置。这可以保留你离开的路径中的重要上下文，而无需重放整个分支。

当被提示时，选择以下之一：

1. 不生成摘要
2. 使用默认提示进行摘要
3. 使用自定义关注点指令进行摘要

有关分支摘要的内部机制和扩展钩子，请参阅[压缩](zh/compaction.md)。

## 会话格式

会话文件是 JSONL 格式，包含消息记录、模型变更、思维层级变更、标签、压缩、分支摘要和扩展记录。

有关解析器、扩展、SDK 用法和完整的 SessionManager API，请参阅[会话格式](zh/session-format.md)。
